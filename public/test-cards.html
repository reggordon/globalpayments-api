General Information

  • All test cards use CVV: 123 (or 1234 for Amex) unless otherwise noted
  • Use any future expiry date (e.g., 12/25, 01/30)
  • Use any name for cardholder name
  • Test cards only work in sandbox environment
  • 3D Secure testing available with specific cards (see below)
  • Note: Global Payments sandbox may approve most cards by default. Use specific CVV values or card numbers below to test declines and errors

3D Secure Testing

  • Use 3DS enrolled cards to test authentication challenges
  • In sandbox, you'll see a test authentication page
  • Any password will work in the test 3DS interface
  • 3DS adds an extra security layer for card-not-present transactions

Testing Tips

  • Amounts: Use any amount - sandbox doesn't process real money
  • Currencies: Test with EUR, USD, GBP, and other supported currencies
  • Multiple Attempts: You can reuse the same card multiple times
  • Transaction History: All test transactions are logged for review
  • Result Codes: Pay attention to response codes for debugging
